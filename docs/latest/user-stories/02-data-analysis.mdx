Querying experiment history and analyzing results.

## Stories

### DATA-001 List Experiment History

**As a** Quantum Researcher,
**I want to** see a list of past experiments,
**So that** I can find results I want to review.

**Acceptance Criteria:**
- [ ] Can list via CLI (`qca history list`)
- [ ] Shows experiment ID, type, timestamp, and status
- [ ] Most recent experiments shown first
- [ ] Can limit number of results

**Status:** Draft

---

### DATA-002 View Experiment Details

**As a** Quantum Researcher,
**I want to** see full details of a specific experiment,
**So that** I can understand what was run and what resulted.

**Acceptance Criteria:**
- [ ] Can view via CLI (`qca history show <id>`)
- [ ] Shows all input parameters
- [ ] Shows scalar results and status
- [ ] Shows file path to raw data

**Status:** Draft

---

### DATA-003 View Experiment Plots

**As a** Quantum Researcher,
**I want to** see the plots generated by an experiment,
**So that** I can visually analyze the data.

**Acceptance Criteria:**
- [ ] Plots display in web UI
- [ ] Can export plots as images
- [ ] Interactive zoom/pan in web UI

**Status:** Draft

---

### DATA-004 Get AI Analysis of Plot

**As a** Quantum Researcher,
**I want to** get an AI interpretation of my experiment plot,
**So that** I can understand anomalies or patterns I might miss.

**Acceptance Criteria:**
- [ ] Can ask agent "analyze this plot"
- [ ] VLM provides interpretation of visual features
- [ ] Identifies potential issues (e.g., poor fit, unexpected peaks)

**Status:** Draft

---

### DATA-005 Access Raw Data Arrays

**As a** Quantum Researcher,
**I want to** access the raw numerical data from an experiment,
**So that** I can do custom analysis in my own tools.

**Acceptance Criteria:**
- [ ] Can list arrays via CLI (`qca data arrays <id>`)
- [ ] Can get specific array (`qca data get <id> <array>`)
- [ ] Data exported in usable format (JSON, CSV, or NumPy-compatible)

**Status:** Draft

---
