## Quantum Researcher

**Role:** Primary user of QCA

**Background:**
- PhD-level physicist or engineer working on quantum computing
- Deep understanding of quantum mechanics and calibration concepts
- Comfortable with Python and command-line tools
- May not be an expert in LLMs or AI agents

**Goals:**
- Quickly calibrate quantum devices without manual scripting
- Understand experiment results through AI-assisted analysis
- Iterate on parameters based on intelligent recommendations
- Document calibration sessions for reproducibility

**Pain Points:**
- Manual calibration is time-consuming and error-prone
- Difficult to remember optimal parameter ranges
- Analyzing plots requires domain expertise
- Tracking experiment history across sessions

**Typical Tasks:**
- Run single calibration experiments with specific parameters
- Analyze spectroscopy and oscillation data
- Compare results across multiple runs
- Get recommendations for next calibration steps

## Lab Technician

**Role:** Operator executing routine calibrations

**Background:**
- Technical background, possibly MS-level
- Follows established protocols and procedures
- Less focus on research, more on operational tasks
- Needs reliable, repeatable processes

**Goals:**
- Execute standard calibration workflows reliably
- Monitor workflow progress and catch failures early
- Generate reports for quality assurance
- Minimize downtime during calibration

**Pain Points:**
- Complex multi-step procedures are easy to mess up
- Hard to know when something goes wrong mid-workflow
- Documentation of completed steps is tedious

**Typical Tasks:**
- Run predefined calibration workflows
- Monitor workflow execution status
- Pause/resume workflows as needed
- Export results for reporting

## System Administrator

**Role:** Manages QCA deployment and configuration

**Background:**
- DevOps or IT background
- Manages lab infrastructure
- May not understand quantum calibration details
- Focused on reliability and security

**Goals:**
- Deploy and maintain QCA instances
- Configure LLM providers and API keys securely
- Monitor system health and usage
- Manage user access and permissions

**Pain Points:**
- Multiple configuration options across providers
- Ensuring API keys are secure
- Debugging connectivity issues

**Typical Tasks:**
- Initial system setup and configuration
- Switch between LLM providers
- Configure custom experiment scripts location
- Monitor server health
