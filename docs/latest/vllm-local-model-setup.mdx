## Model

- **Base**: Qwen 3.5 (fine-tuned)
- **Name**: `35b_fewshot_all_then_singleshot_lr1e5_lr1e5`
- **Architecture**: Qwen 3.5 35B (A3B MoE)

## vLLM Launch Command

```bash
vllm serve <model_path> \
  --enable-auto-tool-choice \
  --tool-call-parser qwen3_coder \
  --reasoning-parser qwen3 \
  --tensor-parallel-size 2 \
  --port 18006
```

### Required Flags

| Flag | Purpose |
|------|---------|
| `--enable-auto-tool-choice` | Enables tool call parsing in vLLM |
| `--tool-call-parser qwen3_coder` | Parses Qwen 3.5's XML-based tool call format |
| `--reasoning-parser qwen3` | Parses thinking/reasoning tokens |

### Qwen 3.5 Tool Call Format

The model outputs tool calls as XML:

```xml
<tool_call>
<function=lab>
<parameter=action>list_experiments</parameter>
</function>
</tool_call>
```

The `qwen3_coder` parser converts this into OpenAI-compatible `tool_calls` JSON responses.

> **Note:** Requires vLLM with [PR #35347](https://github.com/vllm-project/vllm/pull/35347) merged (fixes Qwen 3.5 tool calling template).

## QCA Configuration

`.env` file:

```bash
QCA_MODEL=openai:/home/shuxiangc/fsw/workspace/qwen35_llamafactory/runs/35b_fewshot_all_then_singleshot_lr1e5_lr1e5
OPENAI_BASE_URL=http://localhost:18006/v1
OPENAI_API_KEY=dummy
```

### How It Works

1. `cli.py` loads `.env` via `dotenv`
2. `QCA_MODEL=openai:...` creates a `ChatOpenAI` instance with `use_responses_api=False`
3. `OPENAI_BASE_URL` points the OpenAI client to the local vLLM server
4. vLLM receives requests with OpenAI-compatible tools, model generates XML tool calls, `qwen3_coder` parser converts them to OpenAI tool_calls format

### Differences from Hosted NVIDIA API

| Aspect | NVIDIA API | Local vLLM |
|--------|-----------|------------|
| Prefix | `nvidia:` | `openai:` |
| Base URL | `integrate.api.nvidia.com` | `localhost:18006` |
| Auth | `NVIDIA_API_KEY` | Any value (not checked) |
| Tool calling | Native | Requires `--tool-call-parser qwen3_coder` |
| Model class | `ChatNVIDIA` | `ChatOpenAI` |
