QCA provides a powerful experiment system for quantum calibration and characterization tasks. Experiments are Python scripts that follow a specific structure and can be discovered, validated, and executed through the CLI or agent tools.

## What are Experiments?

Experiments in QCA are Python functions that:
- Accept typed parameters with optional range constraints
- Perform measurements or simulations
- Return structured results including data arrays and plots
- Are automatically discovered from the `scripts/` directory

Unlike traditional scripts, QCA experiments:
- Have their parameters automatically validated
- Are type-safe with runtime checks
- Include metadata for agent reasoning
- Support both CLI and agent-driven execution
- Store results in a queryable format (HDF5 + SQLite)

## Experiment Lifecycle

```
Discovery → Validation → Execution → Storage → Retrieval
    ↓            ↓            ↓           ↓          ↓
 AST Parse   Type Check   Subprocess   HDF5/DB   Query API
```

### 1. Discovery
When you run `qca experiments list`, the system:
- Scans the `scripts/` directory for `.py` files
- Parses each file using Python's AST (no code execution)
- Extracts function signatures, type hints, and docstrings
- Builds an experiment schema for each valid script

### 2. Validation
Before execution, parameters are validated:
- Type checking (int, float, str, bool, list)
- Range validation for `Annotated[type, (min, max)]` parameters
- Required vs optional parameter checks

### 3. Execution
Experiments run in isolated subprocesses:
- Parameters passed as JSON via stdin
- Results returned as JSON via stdout
- Progress messages go to stderr
- Configurable timeout (default 300s)

### 4. Storage
Results are automatically saved:
- HDF5 files: `data/experiments/YYYY/MM/DD/HHMMSS_type.h5`
- SQLite index: `data/index.db` for fast queries
- Includes metadata, arrays, plots, and parameters

### 5. Retrieval
Access stored experiments via:
- `qca history list` - List past experiments
- `qca history show <id>` - View experiment details
- `qca data arrays <id>` - Access array data
- `lab` tool in agent mode - Agent-driven queries

## Available Experiment Types

QCA discovers experiments from the `scripts/` directory. To see what's available:

```bash
qca experiments list
```

Common experiment types include:
- **Spectroscopy**: Frequency sweeps to find qubit transitions
- **Rabi**: Amplitude/time sweeps for pulse calibration
- **T1/T2**: Coherence time measurements
- **Readout**: SNR and fidelity characterization
- **Two-tone**: Qubit-resonator spectroscopy

The actual experiments depend on your project. See [Writing Experiment Scripts](/user-guide/experiments/writing-scripts) to create your own.

## Quick Examples

### List Available Experiments
```bash
qca experiments list
```

### View Experiment Schema
```bash
qca experiments schema qubit_spectroscopy
```

### Run an Experiment
```bash
qca experiments run qubit_spectroscopy \
  --param center_freq=4.8 \
  --param span=0.2 \
  --param num_points=101
```

### View Results
```bash
# List recent experiments
qca history list

# Show details
qca history show <experiment-id>

# Get array data
qca data get <experiment-id> frequencies
```

## Next Steps

Start by running your first experiment:

```bash
# Discover what's available
qca experiments list

# View the schema
qca experiments schema <experiment-name>

# Run it
qca experiments run <experiment-name> --param key=value
```

Or try the agent-driven workflow:

```bash
qca -n "Run qubit spectroscopy centered at 4.85 GHz"
```

The agent will use the `lab` tool to discover, validate, and execute experiments for you.
