QCA uses a dual-storage system combining HDF5 files for data and SQLite for fast indexing and queries.

## Storage Architecture

```
data/
├── index.db                    # SQLite index for fast queries
└── experiments/                # HDF5 files organized by date
    └── YYYY/
        └── MM/
            └── DD/
                ├── HHMMSS_experiment_type.h5
                ├── HHMMSS_experiment_type.h5
                └── ...
```

### Why This Design?

**HDF5 for data:**
- Efficient storage of large arrays
- Self-describing format (metadata + data)
- Industry standard for scientific data
- Supports partial reads (slicing)

**SQLite for index:**
- Fast queries without opening HDF5 files
- Filter by type, date, target
- List recent experiments
- No database server required

## Storage Structure

### Directory Layout

When an experiment completes, it's saved to:

```
data/experiments/YYYY/MM/DD/HHMMSS_type.h5
```

Example:
```
data/experiments/2026/03/23/143052_qubit_spectroscopy.h5
data/experiments/2026/03/23/143128_rabi_amplitude.h5
data/experiments/2026/03/23/150234_qubit_spectroscopy.h5
```

The filename includes:
- **HHMMSS**: Timestamp (hour, minute, second)
- **type**: Experiment name

### HDF5 File Format

Each HDF5 file contains:

```
root/
├── attributes (metadata)
│   ├── id: "exp_20260323_143052_qubit_spectroscopy"
│   ├── type: "qubit_spectroscopy"
│   ├── timestamp: "2026-03-23T14:30:52Z"
│   ├── status: "success"
│   ├── target: "qubit_0" (optional)
│   ├── params: "{\"center_freq\": 4.8, ...}" (JSON string)
│   ├── results: "{\"qubit_freq\": 4.852, ...}" (JSON string)
│   └── notes: "Calibration run after cooldown"
├── arrays/                     # Group containing data arrays
│   ├── frequencies: [4.7, 4.71, 4.72, ...]
│   ├── excited_population: [0.01, 0.02, 0.45, ...]
│   └── ...
└── plots/                      # Group containing plot data
    ├── "Qubit spectroscopy response"
    │   ├── data: <binary blob>
    │   └── format: "plotly"
    └── ...
```

### SQLite Index Schema

The `index.db` SQLite database has one table:

```sql
CREATE TABLE experiments (
    id TEXT PRIMARY KEY,              -- Unique experiment ID
    type TEXT NOT NULL,                -- Experiment name
    target TEXT,                       -- Optional target (e.g., "qubit_0")
    timestamp TEXT NOT NULL,           -- ISO 8601 timestamp
    status TEXT NOT NULL,              -- "success" or "failed"
    params TEXT,                       -- JSON string of parameters
    results TEXT,                      -- JSON string of results
    notes TEXT,                        -- User notes
    file_path TEXT NOT NULL            -- Full path to HDF5 file
);

CREATE INDEX idx_type ON experiments(type);
CREATE INDEX idx_timestamp ON experiments(timestamp);
```

This enables fast queries like:
```sql
-- Find all qubit_spectroscopy experiments
SELECT * FROM experiments WHERE type = 'qubit_spectroscopy';

-- Find last 10 experiments
SELECT * FROM experiments ORDER BY timestamp DESC LIMIT 10;

-- Find experiments on a specific target
SELECT * FROM experiments WHERE target = 'qubit_0';
```

## Accessing Stored Data

### Using CLI Commands

#### List Experiments

```bash
# List all experiments
qca history list

# List last 10
qca history list --last 10

# Filter by type
qca history list --type qubit_spectroscopy

# Filter by target
qca history list --target qubit_0
```

Example output:
```

exp_20260323_143052_qubit_spectroscopy  qubit_spectroscopy   2026-03-23 14:30:52  success
exp_20260323_150234_qubit_spectroscopy  qubit_spectroscopy   2026-03-23 15:02:34  success
```

### Using the lab Tool (Agent Mode)

The lab tool provides programmatic access to stored data:

```python
# List recent experiments
lab.history_list(last=10)

# Show specific experiment
lab.history_show(experiment_id="exp_20260323_143052_qubit_spectroscopy")

# List arrays
lab.list_arrays(experiment_id="exp_...")
```

### Programmatic Access (Python)

You can access the storage layer directly:

```python
from pathlib import Path
from lab.storage import (
    load_experiment,
    search_experiments,
    list_arrays,
    get_array,
    get_array_stats
)

data_dir = Path("data")

# Search experiments
experiments = search_experiments(
    data_dir,
    type="qubit_spectroscopy",
    last=10
)

for exp in experiments:
    print(f"{exp['id']}: {exp['timestamp']}")

# Load full experiment
result = load_experiment("exp_...", data_dir)
print(result.params)
print(result.results)
print(result.arrays)
print(result.plots)

# Access arrays
arrays = list_arrays("exp_...", data_dir)
freq_data = get_array("exp_...", "frequencies", data_dir)
stats = get_array_stats("exp_...", "frequencies", data_dir)
```

## Data Retention and Management

### Deleting Experiments

```bash
qca history delete <experiment-id>
```

This:
1. Deletes the HDF5 file
2. Removes the entry from SQLite index

**Warning:** This is permanent and cannot be undone.

### Reindexing

If the SQLite index becomes corrupted or out of sync:

```bash
qca history reindex
```

This:
1. Clears the SQLite index
2. Scans all HDF5 files in `data/experiments/`
3. Rebuilds the index from HDF5 metadata

Use this if:
- SQLite database is deleted
- HDF5 files are manually moved
- Index reports missing files

### Backup Strategies

**Option 1: Backup entire data directory**
```bash
tar -czf data-backup-$(date +%Y%m%d).tar.gz data/
```

**Option 2: Backup specific date range**
```bash
tar -czf data-backup-202603.tar.gz data/experiments/2026/03/
```

**Option 3: Export to external storage**
```bash
rsync -av data/experiments/ /mnt/external/qca-data/
```

### Archiving Old Experiments

Move old experiments to archive:
```bash
mkdir -p archive/2026/
mv data/experiments/2026/01/ archive/2026/
qca history reindex  # Update index
```

## Storage Efficiency

### File Sizes

Typical HDF5 file sizes:
- Metadata only: ~5 KB
- Small experiment (100 points): ~10 KB
- Medium experiment (1000 points): ~50 KB
- Large experiment (10000 points): ~500 KB
- With plots: Add 50-500 KB per plot

### Compression

HDF5 files are not compressed by default, but you can enable compression:

```python
# In experiment script, if using h5py directly
import h5py

with h5py.File("output.h5", "w") as f:
    f.create_dataset("data", data=array, compression="gzip", compression_opts=9)
```

QCA's storage layer doesn't enable compression by default for speed, but you can modify `core/storage.py` if needed.

### Partial Reads

HDF5 supports reading slices without loading the entire array:

```bash
# Only reads elements 0-9
qca data get <experiment-id> large_array --start 0 --end 10
```

This is efficient for large arrays (millions of points).

## Data Format Details

### Timestamp Format

Timestamps are stored in ISO 8601 format with UTC timezone:
```
2026-03-23T14:30:52Z
```

Parse in Python:
```python
from datetime import datetime
dt = datetime.fromisoformat(timestamp.replace("Z", "+00:00"))
```

### Parameter and Result Encoding

Parameters and results are stored as JSON strings in HDF5 attributes:

```python
params = {"center_freq": 4.8, "span": 0.2}
f.attrs["params"] = json.dumps(params)
```

This allows:
- Nested dictionaries
- Lists
- Mixed types
- Easy serialization

### Array Storage

Arrays are stored as HDF5 datasets:

```python
# NumPy arrays
frequencies = np.array([4.7, 4.71, 4.72, ...])
f["arrays/frequencies"] = frequencies

# Python lists are converted
populations = [0.01, 0.02, 0.03, ...]
f["arrays/populations"] = np.array(populations)
```

Supported dtypes:
- int, float (various precisions)
- bool
- string (as h5py fixed-length strings)

### Plot Storage

Plots are stored as binary blobs with format metadata:

```python
# Plotly JSON
plot_json = json.dumps(plotly_data)
plot_bytes = plot_json.encode("utf-8")
dataset = f.create_dataset("plots/my_plot", data=np.frombuffer(plot_bytes, dtype=np.uint8))
dataset.attrs["format"] = "plotly"

# PNG image
png_base64 = "iVBORw0KG..."
png_bytes = png_base64.encode("utf-8")
dataset = f.create_dataset("plots/my_image", data=np.frombuffer(png_bytes, dtype=np.uint8))
dataset.attrs["format"] = "png"
```

## Inspecting HDF5 Files

### Using h5dump (CLI)

```bash
# View file structure
h5dump -n data/experiments/2026/03/23/143052_qubit_spectroscopy.h5

# View metadata
h5dump -a id data/experiments/2026/03/23/143052_qubit_spectroscopy.h5

# View dataset
h5dump -d /arrays/frequencies data/experiments/2026/03/23/143052_qubit_spectroscopy.h5
```

### Using h5py (Python)

```python
import h5py

with h5py.File("data/experiments/2026/03/23/143052_qubit_spectroscopy.h5", "r") as f:
    # View attributes
    print(f.attrs["id"])
    print(f.attrs["type"])

    # List groups
    print(list(f.keys()))

    # Access arrays
    frequencies = f["arrays/frequencies"][:]
    print(frequencies)

    # View array metadata
    print(f["arrays/frequencies"].shape)
    print(f["arrays/frequencies"].dtype)
```

### Using HDFView (GUI)

Download HDFView from https://www.hdfgroup.org/downloads/hdfview/

Open an `.h5` file to browse:
- View metadata as attributes
- Inspect array shapes and types
- Plot arrays

## Best Practices

### 1. Use Meaningful Targets

```python
# Good
return {"status": "success", "target": "qubit_0"}

# Also good
return {"status": "success", "target": "resonator_A"}
```

This allows filtering:
```bash
qca history list --target qubit_0
```

### 2. Include Notes

```python
return {
    "status": "success",
    "notes": "Post-cooldown calibration run"
}
```

### 3. Keep Arrays Separate from Scalars

```python
# Good - scalars and arrays clearly separated
"data": {
    "qubit_freq": {"type": "scalar", "value": 4.85, ...},
    "frequencies": {"type": "array", "value": [...], ...}
}
```

### 4. Use Consistent Units

```python
# Good - always use GHz for frequency
"unit": "GHz"

# Avoid - mixing units
"unit": "MHz"  # in one experiment
"unit": "GHz"  # in another
```

### 5. Test Serialization

```python
# Always test that arrays are JSON-serializable
import numpy as np
arr = np.array([1, 2, 3])
serializable = arr.tolist()  # Convert to list
```

## Troubleshooting

### Missing Experiments

If an experiment doesn't appear in `qca history list`:

```bash
# Reindex from HDF5 files
qca history reindex
```

### Corrupted HDF5 Files

Check file integrity:
```bash
h5dump -n file.h5
```

If corrupted, the file cannot be recovered. Regular backups are essential.

### SQLite Lock Errors

If you see "database is locked":
- Close other processes accessing `index.db`
- Wait a moment and retry
- In rare cases, delete `index.db` and reindex
