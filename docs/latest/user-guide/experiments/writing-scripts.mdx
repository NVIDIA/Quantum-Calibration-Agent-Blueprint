This guide covers how to create custom experiment scripts for QCA.

## Script Location

For security reasons, the QCA agent only has access to files within the project directory. To use a new experiment script, you must first copy it into the `scripts/` folder at the project root (e.g., `<qca-project-root>/scripts/`). Scripts located outside the project directory cannot be accessed or executed by the agent.

```bash
# Copy your script to the scripts folder
cp /path/to/your/experiment.py <qca-project-root>/scripts/
```

## Script Requirements

QCA experiments are Python scripts with specific requirements:

1. **File naming**: Must be a `.py` file without underscore prefix
2. **Public function**: Must contain at least one public function (no `_` prefix)
3. **Return type**: Function must have `-> dict` return type annotation
4. **Type hints**: Parameters must have type annotations
5. **Return structure**: Must return a dict with specific keys

## Minimal Example

```python
from typing import Annotated

def simple_measurement(
    frequency: Annotated[float, (4.0, 6.0)] = 5.0
) -> dict:
    """Measure something at a frequency.

    Args:
        frequency: Center frequency in GHz (range: 4.0-6.0)
    """
    result = frequency * 2

    return {
        "status": "success",
        "data": {
            "result": {
                "type": "scalar",
                "value": result,
                "unit": "MHz",
                "name": "Doubled frequency"
            }
        }
    }
```

This script:
- Has a public function `simple_measurement`
- Uses `Annotated[float, (min, max)]` for range validation
- Returns `-> dict`
- Returns required `status` field

## Function Signature Requirements

### Return Type Annotation

**Required**: The function must have `-> dict` return type.

```python
# Good
def my_experiment(...) -> dict:
    ...

# Bad - no return type
def my_experiment(...):
    ...

# Bad - wrong return type
def my_experiment(...) -> str:
    ...
```

### Parameter Type Annotations

All parameters you want QCA to handle must have type annotations.

**Supported types:**
- `int` - Integers
- `float` - Floating-point numbers
- `str` - Strings
- `bool` - Booleans
- `list` - Lists

```python
def my_experiment(
    count: int,
    frequency: float,
    target: str,
    verbose: bool,
    amplitudes: list
) -> dict:
    ...
```

### Optional vs Required Parameters

Parameters with default values are optional. Parameters without defaults are required.

```python
def my_experiment(
    required_param: float,              # Required
    optional_param: float = 5.0,        # Optional
    another_optional: int = 100         # Optional
) -> dict:
    ...
```

When calling:
```bash
# Must provide required_param
qca experiments run my_experiment --param required_param=4.5

# Can override optional_param
qca experiments run my_experiment \
  --param required_param=4.5 \
  --param optional_param=6.0
```

## Using Annotated for Parameter Ranges

Use `typing.Annotated` to specify valid ranges for numeric parameters.

```python
from typing import Annotated

def qubit_spectroscopy(
    center_freq: Annotated[float, (4.0, 5.5)] = 4.8,
    num_points: Annotated[int, (51, 201)] = 101
) -> dict:
    """Perform qubit spectroscopy.

    Args:
        center_freq: Center frequency in GHz (4.0-5.5)
        num_points: Number of frequency points (51-201)
    """
    ...
```

Format: `Annotated[type, (min, max)]`
- First element: The type (int, float)
- Second element: Tuple of (min, max) bounds

**Benefits:**
- Automatic validation before execution
- Agent can reason about valid values
- Schema includes range information
- Prevents invalid parameter errors

## Required Output Format

The function must return a dictionary with the following structure:

### Minimum Required Structure

```python
{
    "status": "success"  # or "failed"
}
```

### Full Structure

```python
{
    "status": "success",           # Required: "success" or "failed"
    "error": "...",                # Required if status is "failed"
    "data": {                      # Optional: scalar values and arrays
        "scalar_name": {
            "type": "scalar",
            "value": 4.85,
            "unit": "GHz",
            "name": "Human-readable name"
        },
        "array_name": {
            "type": "array",
            "value": [1, 2, 3, ...],
            "unit": "V",
            "name": "Human-readable name"
        }
    },
    "plots": [                     # Optional: list of plots
        {
            "name": "Plot title",
            "format": "plotly",    # or "png"
            "data": {...}          # Plotly JSON or base64 PNG
        }
    ]
}
```

### Status Field

**Required field.** Must be either `"success"` or `"failed"`.

```python
# Success
return {"status": "success", ...}

# Failure
return {"status": "failed", "error": "Calibration not found"}
```

### Data Field

The `data` field contains measurement results. Each entry has:
- `type`: Either `"scalar"` or `"array"`
- `value`: The actual data
- `unit`: Physical unit (e.g., "GHz", "V", "probability")
- `name`: Human-readable description

**Scalar example:**
```python
"data": {
    "qubit_freq": {
        "type": "scalar",
        "value": 4.852,
        "unit": "GHz",
        "name": "Fitted qubit transition frequency"
    },
    "snr": {
        "type": "scalar",
        "value": 42.3,
        "unit": "dimensionless",
        "name": "Signal-to-noise ratio"
    }
}
```

**Array example:**
```python
"data": {
    "frequencies": {
        "type": "array",
        "value": [4.7, 4.71, 4.72, ...],
        "unit": "GHz",
        "name": "Drive frequencies"
    },
    "populations": {
        "type": "array",
        "value": [0.01, 0.02, 0.45, ...],
        "unit": "probability",
        "name": "Excited state population"
    }
}
```

### Plots Field

The `plots` field is a list of plot dictionaries. Each plot has:
- `name`: Plot title
- `format`: `"plotly"` (recommended) or `"png"`
- `data`: Plot data (Plotly JSON dict or base64 string)

**Plotly format (recommended):**
```python
"plots": [
    {
        "name": "Qubit spectroscopy response",
        "format": "plotly",
        "data": {
            "data": [{
                "x": frequencies,
                "y": populations,
                "type": "scatter",
                "mode": "lines+markers",
                "name": "Population"
            }],
            "layout": {
                "title": "Qubit Spectroscopy",
                "xaxis": {"title": "Frequency (GHz)"},
                "yaxis": {"title": "Excited State Population"}
            }
        }
    }
]
```

**PNG format:**
```python
import base64
import matplotlib.pyplot as plt
from io import BytesIO

# Create plot
fig, ax = plt.subplots()
ax.plot(x, y)

# Convert to base64
buf = BytesIO()
fig.savefig(buf, format='png')
buf.seek(0)
png_base64 = base64.b64encode(buf.read()).decode('utf-8')

return {
    "status": "success",
    "plots": [{
        "name": "My plot",
        "format": "png",
        "data": png_base64
    }]
}
```

## Complete Example

Here's a complete experiment script with all features:

```python
"""Qubit spectroscopy experiment with realistic simulation."""

import time
from typing import Annotated
import numpy as np

def qubit_spectroscopy(
    center_freq: Annotated[float, (4.0, 5.5)] = 4.8,
    span: Annotated[float, (0.05, 0.5)] = 0.2,
    num_points: Annotated[int, (51, 201)] = 101,
    drive_amplitude: Annotated[float, (0.01, 1.0)] = 0.1,
    num_averages: Annotated[int, (500, 20000)] = 5000,
) -> dict:
    """Perform qubit spectroscopy measurement.

    Sweeps drive frequency to find the qubit transition frequency.

    Args:
        center_freq: Center frequency of sweep (GHz)
        span: Frequency span of sweep (GHz)
        num_points: Number of frequency points
        drive_amplitude: Drive pulse amplitude (a.u.)
        num_averages: Number of measurement averages

    Returns:
        Result dictionary with frequencies, populations, and fitted frequency
    """
    # Simulate execution time
    time.sleep(2.0)

    # Generate frequency sweep
    frequencies = np.linspace(
        center_freq - span / 2,
        center_freq + span / 2,
        num_points
    )

    # Simulate qubit response (Lorentzian)
    true_freq = 4.85  # GHz
    linewidth = 0.01  # GHz
    delta = frequencies - true_freq
    population = 0.8 * linewidth**2 / (delta**2 + linewidth**2)

    # Add noise
    noise = np.random.normal(0, 0.02, num_points)
    population_noisy = np.clip(population + noise, 0, 1)

    # Find peak
    peak_idx = np.argmax(population_noisy)
    fitted_freq = frequencies[peak_idx]
    peak_value = population_noisy[peak_idx]

    # Calculate SNR
    snr = peak_value / 0.02

    return {
        "status": "success",
        "data": {
            "frequencies": {
                "type": "array",
                "value": frequencies.tolist(),
                "unit": "GHz",
                "name": "Drive frequencies",
            },
            "excited_population": {
                "type": "array",
                "value": population_noisy.tolist(),
                "unit": "probability",
                "name": "Excited state population",
            },
            "qubit_freq": {
                "type": "scalar",
                "value": float(fitted_freq),
                "unit": "GHz",
                "name": "Fitted qubit transition frequency",
            },
            "snr": {
                "type": "scalar",
                "value": float(snr),
                "unit": "dimensionless",
                "name": "Signal-to-noise ratio",
            },
        },
        "plots": [
            {
                "name": "Qubit spectroscopy response",
                "format": "plotly",
                "data": {
                    "data": [{
                        "x": frequencies.tolist(),
                        "y": population_noisy.tolist(),
                        "type": "scatter",
                        "mode": "lines+markers",
                        "name": "Population",
                        "marker": {"size": 4},
                    }],
                    "layout": {
                        "title": f"Qubit Spectroscopy - f_01 = {fitted_freq:.4f} GHz",
                        "xaxis": {"title": "Drive Frequency (GHz)"},
                        "yaxis": {"title": "Excited State Population", "range": [0, 1]},
                    },
                },
            }
        ],
    }
```

## Progress Output

Experiments can print progress messages to `stderr`. These are shown to the user but don't interfere with the JSON output on `stdout`.

```python
import sys

def _log(msg):
    """Print to stderr for progress updates."""
    print(msg, file=sys.stderr)

def my_experiment(param: float) -> dict:
    """Experiment with progress output."""
    _log("=" * 60)
    _log("EXPERIMENT - Starting")
    _log("=" * 60)
    _log(f"  Parameter: {param}")
    _log("-" * 60)

    # Do work...
    _log("  [12:34:56] Acquiring data...")
    time.sleep(5)

    _log("  [12:35:01] Complete")
    _log("=" * 60)

    return {"status": "success"}
```

Output appears during execution:
```
===========================================================
EXPERIMENT - Starting
===========================================================
  Parameter: 5.0
-----------------------------------------------------------
  [12:34:56] Acquiring data...
  [12:35:01] Complete
===========================================================
```

## Validation

Before using a script, validate it:

```bash
qca experiments validate scripts/my_experiment.py
```

This performs comprehensive checks:
- File exists and is `.py`
- Not a private file (no `_` prefix)
- Python syntax is valid
- Has at least one public function
- Function has `-> dict` return type
- Function has typed parameters
- Checks for docstring (warning if missing)
- Lists parameters with ranges

## Best Practices

### 1. Use Descriptive Names

```python
# Good
def qubit_spectroscopy(...)
def rabi_amplitude_sweep(...)
def t1_relaxation(...)

# Bad
def exp1(...)
def test(...)
def run(...)
```

### 2. Include Docstrings

```python
def my_experiment(...) -> dict:
    """Short one-line description.

    Longer description explaining what the experiment does,
    what it measures, and any important details.

    Args:
        param1: Description of param1
        param2: Description of param2

    Returns:
        Dictionary with status, data, and plots
    """
```

### 3. Use Annotated Ranges

```python
# Good - validates parameters
frequency: Annotated[float, (4.0, 5.5)]

# OK - but no automatic validation
frequency: float
```

### 4. Include Units in Data

```python
# Good
"value": 4.85,
"unit": "GHz"

# Bad - ambiguous
"value": 4850000000
```

### 5. Provide Informative Names

```python
# Good
"name": "Fitted qubit transition frequency"

# Bad
"name": "Result 1"
```

### 6. Handle Errors Gracefully

```python
def my_experiment(...) -> dict:
    try:
        # Do measurement
        result = perform_measurement()

        return {
            "status": "success",
            "data": {...}
        }
    except Exception as e:
        return {
            "status": "failed",
            "error": f"Measurement failed: {str(e)}"
        }
```

### 7. Use Plotly for Interactive Plots

Plotly plots are:
- Interactive (zoom, pan, hover)
- Stored as JSON (no image rendering needed)
- Easy to customize
- Preferred by agents for data extraction

## Common Patterns

### Multiple Measurements

```python
def multi_qubit_spectroscopy(
    qubits: list = [0, 1, 2]
) -> dict:
    """Measure multiple qubits."""
    frequencies = {}

    for qubit_id in qubits:

    return {
        "status": "success",
        "data": frequencies
```

```python
def adaptive_measurement(
    initial_guess: float,
    tolerance: float = 0.01
```

## Troubleshooting

### Script Not Discovered

**Problem:** Script doesn't appear in `qca experiments list`

**Check:**
- File is in `scripts/` directory
- File has `.py` extension
- File doesn't start with `_`
- Function doesn't start with `_`
- Function has `-> dict` return type
- At least one parameter has type annotation

### Validation Errors

**Problem:** `qca experiments validate` reports errors

**Solutions:**
- Add `-> dict` return type annotation
- Add type hints to parameters
- Fix Python syntax errors
- Add at least one public function

### Runtime Errors

**Problem:** Experiment fails during execution

**Check:**
- All dependencies are installed
- Return dictionary has `status` field
- Arrays are JSON-serializable (use `.tolist()` for numpy arrays)
- No print statements to stdout (use stderr)
